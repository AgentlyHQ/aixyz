## Overview

`create-aixyz-app` is a project scaffolding CLI that generates a new aixyz agent project with the standard directory structure, configuration, and example files.

## Usage

```bash
bunx create-aixyz-app my-agent
```

The CLI prompts for:

- **Agent name** — Used in `aixyz.config.ts` and `package.json`

The boilerplate uses [`@browser-ai/transformers-js`](https://github.com/jakobhoeg/browser-ai) with [`@huggingface/transformers`](https://huggingface.co/docs/transformers.js/en/index) (Transformers.js v3) to run `HuggingFaceTB/SmolLM2-1.7B-Instruct` entirely in-process via WebAssembly — no external server or API key required.

### Non-Interactive Mode

Skip all prompts with the `-y` flag:

```bash
bunx create-aixyz-app my-agent -y
```

## What Gets Scaffolded

```
my-agent/
  aixyz.config.ts       # Agent config (name, description, skills)
  app/
    agent.ts            # Agent definition using Vercel AI SDK
    tools/
      weather.ts        # Example tool
    icon.png            # Agent icon
  package.json          # Dependencies and scripts
  tsconfig.json         # TypeScript configuration
  vercel.json           # Vercel deployment config
  .env.local            # Environment variables
```

### Scripts

The generated `package.json` includes:

```json
{
  "scripts": {
    "dev": "aixyz dev",
    "build": "aixyz build"
  }
}
```

## After Scaffolding

```bash
cd my-agent
bun install
bun run dev
```

The model (`HuggingFaceTB/SmolLM2-1.7B-Instruct`) is downloaded automatically from HuggingFace Hub on first run and cached locally. No external server is required.

Your agent starts at `http://localhost:3000` with A2A, MCP, and x402 support ready to go.
