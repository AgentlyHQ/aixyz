## Vercel

aixyz has first-class Vercel support using the [Build Output API v3](https://vercel.com/docs/build-output-api/3).

<Steps>
  <Step title="Build for Vercel">
    ```bash
    VERCEL=1 aixyz build
    # Or: aixyz build --output vercel
    ```
    This generates the `.vercel/output/` directory in the required format.
  </Step>
  <Step title="Configure vercel.json">
    The scaffolded project includes a `vercel.json`. Ensure it points to your build output:
    ```json
    {
      "buildCommand": "aixyz build",
      "outputDirectory": ".vercel/output"
    }
    ```
  </Step>
  <Step title="Deploy">
    ```bash
    vercel --prod
    ```
  </Step>
</Steps>

Set your environment variables (e.g., `OPENAI_API_KEY`) in the Vercel dashboard under **Settings â†’ Environment Variables**.

## Standalone

The default build output is a single JavaScript file that runs directly with Bun.

```bash
aixyz build
bun .aixyz/output/server.js
```

The server starts on the port specified by the `PORT` environment variable (defaults to `3000`).

## Docker

Create a `Dockerfile` to containerize your standalone build:

```dockerfile
FROM oven/bun:1.3

WORKDIR /app
COPY package.json bun.lock ./
RUN bun install --production

COPY . .
RUN bun run build

EXPOSE 3000
CMD ["bun", ".aixyz/output/server.js"]
```

Build and run:

```bash
docker build -t my-agent .
docker run -p 3000:3000 -e OPENAI_API_KEY=sk-... my-agent
```

<Warning>Never bake API keys into Docker images. Pass them as environment variables at runtime.</Warning>

## Verifying Your Deployment

After deploying, verify your agent is accessible:

```bash
# Check the agent card
curl https://your-agent.example.com/.well-known/agent-card.json

# Test the A2A endpoint
curl -X POST https://your-agent.example.com/agent \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"tasks/send","id":"1","params":{"id":"task-1","message":{"role":"user","parts":[{"type":"text","text":"Hello"}]}}}'
```
